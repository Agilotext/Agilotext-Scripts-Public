General Principles

  • Main root URL is https://api.agilotext.com/api/v1

  • APIs are SSL only.

  • Data download: all data are received from server using only JSON format.

Computation of the token

The value of the token is:

String token = SHA1(username + SECRET_VALUE + edition)

Where SECRET_VALUE is an integer to be chosen, common on main Web site www.agilotext and API api.agilotext.com site.

Google Drive API (Meets Recordings)

The Google Drive API allows to access all users who have defined a shared Meets Recorings folderid.

In order to know which folder id of which username, the user must set the URL of his Meets Recordings with the setGoogleDriveUrlAPI, info will be passed and stored by the back end.

A cron is started each minute and scans all the Meets Recorings folders for the defined usernames:

  • Each video file is then downloaded to Agilotext server.

  • Each video file is sent has a transcript, just as it would have been with the interface.

The API call should be in the user raccount info page on the front end. Calls may be repeated.

How it works

The call will test that the Webhook is OK. The Webhook must be generated with the URL. The test will generate the Webhook on Make or Zapier site, with the button "Redetemine data structure".

Agilottext will thus call the webhookUrl endpoint service with the parameters:
