Démos embed — homepage Agilotext

Capture (entrée audio) et Résultat (transcription, compte rendu, assistant IA) ; optionnellement Espace travail (liste des transcriptions / pipeline) pour montrer la continuité après ingestion. Utilisez les pages ci-dessous en embed Webflow ou ouvrez-les pour prévisualisation.

capture.html — INPUT result.html — OUTPUT workspace-demo.html — liste

Narratif homepage (recommandé)

  1. Bloc titre + courte phrase valeur.
  2. Iframe Capture — « Comment vous nous envoyez l’audio ».
  3. Phrase de liaison du type : Une fois le fichier traité, voici ce que vous obtenez dans Agilotext.
  4. Iframe Résultat — transcription audio + livrables.
  5. (Optionnel) Iframe Espace travail — file d’attente / historique pour souligner que rien ne se perd après upload.

Checklist QA avant mise en ligne

  • Lecture audio OK ; durée affichée ~30 s ; segments surlignés pendant la lecture.
  • Clic sur les horodatages du transcript → saut dans l’audio.
  • Onglets accessibles au clavier (← / →, Home / End).
  • Puces Assistant IA : deux interactions statiques, pas de requête réseau.
  • Pas de scroll involontaire sur Capture en desktop (hauteur iframe suffisante).
  • Espace travail : liste lisible en mobile (pile sidebar / contenu).
  • Grep sécurité : pas de token, jobId, api interne dans les fichiers embed.
